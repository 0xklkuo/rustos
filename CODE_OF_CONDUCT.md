# Code of Conduct

## Our Commitment

We want `rustos` to be a welcoming, respectful, and constructive open-source project.

This project exists to help people learn Rust systems programming and modern operating system fundamentals through a small, real codebase. That goal depends on a community where people can ask questions, share ideas, make mistakes, and improve together.

We are committed to making participation in this project a harassment-free experience for everyone, regardless of:

- age
- body size
- disability
- ethnicity
- gender identity or expression
- level of experience
- education
- nationality
- personal appearance
- race
- religion
- sexual identity or orientation

## Our Standards

Examples of behavior that help create a positive environment include:

- being respectful and patient
- using clear, constructive language
- assuming good intent while still giving honest feedback
- focusing on the code, design, and documentation rather than the person
- welcoming questions from learners and new contributors
- accepting feedback gracefully
- explaining technical decisions in plain language when possible
- helping keep discussions focused, teachable, and useful

Examples of unacceptable behavior include:

- harassment, intimidation, or discrimination
- insulting, demeaning, or hostile comments
- personal attacks
- trolling or deliberately disruptive behavior
- repeated bad-faith arguments
- publishing someone else's private information without permission
- sexualized language or imagery
- threats of violence or self-harm
- dismissive gatekeeping based on experience level or background

## Project-Specific Expectations

Because `rustos` is an educational systems project, we ask contributors to be especially mindful of the following:

- prefer teaching over showing off
- prefer clarity over cleverness in communication
- do not shame people for asking beginner questions
- do not use complexity as a status signal
- keep technical disagreement grounded in project goals, constraints, and evidence
- remember that contributors may come from different levels of Rust, OS, and low-level programming experience

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.

Maintainers may remove, edit, or reject comments, commits, code, issues, discussions, pull requests, and other contributions that are not aligned with this Code of Conduct.

Maintainers will aim to act fairly, consistently, and in proportion to the situation.

## Scope

This Code of Conduct applies in all project spaces, including:

- GitHub issues
- pull requests
- discussions
- code review
- documentation review
- project-related chat or communication channels
- public spaces when someone is representing the project

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainer.

For now, use the contact method or reporting path documented in `SUPPORT.md` or the repository profile if available.

When reporting, include as much relevant context as you can, such as:

- what happened
- where it happened
- links or screenshots if relevant
- any other information that may help review the situation

We will review reports in good faith and handle them as carefully as practical.

## Enforcement Guidelines

Possible maintainer responses may include:

- a private warning
- a request to stop the behavior
- removal of content
- temporary suspension from project spaces
- permanent ban from project spaces

The exact response will depend on the severity and pattern of behavior.

## Good-Faith Participation

Not every disagreement or blunt technical comment is a Code of Conduct violation.

This project values direct, honest engineering discussion. The standard is not "always agreeable." The standard is:

- respectful
- constructive
- relevant
- not abusive

## Attribution

This Code of Conduct is informed by the Contributor Covenant and adapted to fit the educational and technical goals of `rustos`.
